## Опис реалізації

## `BouquetItems` Абстрактний базовий клас для всіх елементів букета.
* Поля: `name` (назва), `price` (ціна).
* Методи: геттери та сеттери.

## Flower Базовий клас для квітів.
* Поля: `stemLength` (довжина стебла), `freshness` (свіжість).
* Методи: геттери, сеттери.

## `Rose / Tulip` Конкретні класи квітів-нащадків.

* `Rose` додатково має `hawthorns` (наявність колючок).

* `Tulip` має `budShape` (форма бутона).

## Accessory Клас для аксесуарів букета.
* Поля: `material` (матеріал), `color` (колір).
* Має конструктори за замовчуванням та з параметрами.

## `Bouquet` клас для роботи з букетом:

* Створення масиву `BouquetItems[]`
* Обчислення загальної вартості `getTotalPrice()`
* Сортування квітів за свіжістю `freshnessSort()`
* Пошук квітки за довжиною стебла `findFlower()`

## `Main` для демонстрації роботи:

* Створення квітів і аксесуарів

* Збір букета та підрахунок вартості

* Сортування та пошук квітів
